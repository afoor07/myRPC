# myRPC Client Module

Клиентская часть системы удаленного вызова команд myRPC.

## Назначение

Консольная утилита `myRPC-client` предназначена для отправки команд на удаленный сервер myRPC-server. Клиент автоматически определяет имя текущего пользователя, формирует JSON-запрос и отправляет его через сокет (TCP или UDP) на сервер.

## Файлы модуля

- `client.c` - основной исходный код клиента
- `../include/protocol.h` - заголовочный файл с определениями протокола
- `../include/config.h` - заголовочный файл с конфигурационными константами

## Использование

### Синтаксис

```bash
myRPC-client -c "КОМАНДА" [ОПЦИИ]
```

### Опции командной строки

| Опция | Длинная форма | Описание | Значение по умолчанию |
|-------|---------------|----------|----------------------|
| `-c` | `--command` | Bash команда для выполнения | **Обязательный параметр** |
| `-h` | `--host` | IP-адрес сервера | `localhost` |
| `-p` | `--port` | Порт сервера | `1234` |
| `-s` | `--stream` | Использовать TCP (потоковый) сокет | **По умолчанию** |
| `-d` | `--dgram` | Использовать UDP (датаграммный) сокет | - |
| `-v` | `--version` | Показать версию программы | - |
| `--help` | - | Показать справку | - |

### Примеры использования

#### 1. Базовая команда (localhost, TCP)

```bash
myRPC-client -c "whoami"
```

**Ожидаемый вывод:**
```
root
```

#### 2. Удаленный сервер с явным указанием порта

```bash
myRPC-client -c "date" -h 192.168.1.100 -p 1234 -s
```

**Ожидаемый вывод:**
```
Пт мая 15 10:43:39 MSK 2026
```

#### 3. Команда с аргументами

```bash
myRPC-client -c "ls -la /home" -h 192.168.1.100
```

#### 4. Использование UDP сокета

```bash
myRPC-client -c "uname -a" -h 192.168.1.100 -d
```

#### 5. Команда, вызывающая ошибку

```bash
myRPC-client -c "non_existent_command"
```

**Ожидаемый вывод:**
```
Error: command not found
```

#### 6. Запрещенный пользователь

```bash
myRPC-client -c "whoami"
```

**Ожидаемый вывод (если пользователь не в /etc/myRPC/users.conf):**
```
User 'username' not authorized
```

## Возвращаемые коды

| Код | Значение |
|-----|----------|
| 0 | Успешное выполнение команды |
| 1 | Ошибка (неверные параметры, проблемы с сетью, ошибка авторизации и т.д.) |

## Формат протокола

Клиент отправляет JSON-запрос следующего формата:

```json
{"login":"имя_пользователя","command":"команда bash"}
```

Сервер отвечает JSON-ответом:

```json
{"code":0,"result":"результат выполнения команды"}
```

или в случае ошибки:

```json
{"code":1,"result":"описание ошибки"}
```

## Отладка

### Проверка отправляемых данных с использованием netcat

```bash
# Для TCP
echo '{"login":"root","command":"whoami"}' | nc localhost 1234

# Для UDP
echo '{"login":"root","command":"whoami"}' | nc -u localhost 1234
```

### Просмотр сетевого трафика

```bash
# Мониторинг TCP соединений
sudo tcpdump -i lo port 1234

# Мониторинг UDP
sudo tcpdump -i lo udp port 1234
```

## Обработка ошибок

Клиент обрабатывает следующие ошибки:

1. **Ошибки подключения**: `Connection refused` - сервер не запущен
2. **Таймаут**: Сервер не отвечает (для UDP)
3. **Некорректный JSON ответ**: Сервер вернул невалидный JSON
4. **Ошибка разбора аргументов**: Неверно указаны параметры командной строки

## Зависимости

- **libc6** - стандартная библиотека C
- **getopt** - для разбора аргументов командной строки (встроен в glibc)

## Установка из DEB-пакета

```bash
sudo dpkg -i myrpc-client_1.0_amd64.deb
```

После установки клиент будет доступен как системная команда `myRPC-client`.

## Удаление

```bash
# Удаление пакета
sudo dpkg -r myrpc-client

# Полное удаление с конфигурацией
sudo dpkg --purge myrpc-client
```

## Пример интеграции в скрипты

```bash
#!/bin/bash

# Выполнение команды на удаленном сервере
RESULT=$(myRPC-client -c "df -h" -h 192.168.1.100 -p 1234)

if [ $? -eq 0 ]; then
    echo "Команда выполнена успешно:"
    echo "$RESULT"
else
    echo "Ошибка выполнения команды: $RESULT"
    exit 1
fi
```

## Известные ограничения

1. **Размер команды**: Ограничен размером буфера сокета (обычно 64KB)
2. **Таймаут**: Для UDP-сокетов нет встроенного таймаута (может зависнуть)
3. **Экранирование**: Специальные символы в команде должны быть экранированы оболочкой
